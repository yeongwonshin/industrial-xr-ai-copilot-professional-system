# 영업 플레이북

## 1차 미팅 질문

- 가장 자주 발생하는 작업 오류는 무엇인가요?
- 신입 작업자가 독립 수행하기까지 얼마나 걸리나요?
- 최근 near-miss나 안전 경고 사례는 무엇인가요?
- 매뉴얼/SOP는 작업 중 실제로 얼마나 사용되나요?
- 작업 증거와 감사 대응은 어떻게 하고 있나요?

## 데모 시나리오

1. 작업자가 장비 앞에 선다.
2. QR/객체 인식으로 장비를 식별한다.
3. “이 에러코드 원인이 뭐야?”라고 묻는다.
4. AI가 매뉴얼과 정비이력을 근거로 원인 후보를 제시한다.
5. 작업자가 위험 순서를 시도하면 시스템이 BLOCK한다.
6. PPE/밸브/압력 evidence를 저장한다.
7. 관리자 콘솔에서 작업 로그와 KPI를 확인한다.

## 반대 의견 대응

| 반대 | 대응 |
|---|---|
| 헤드셋 착용이 불편하다 | 작업 시간/환경별 디바이스 선택, 짧은 고위험 절차부터 적용 |
| AI를 믿을 수 없다 | 근거 문서, confidence, 사람 승인, shadow mode 제공 |
| 보안이 걱정된다 | edge/on-prem, redaction, 원본 미저장, RBAC/SIEM 제공 |
| 기존 시스템과 중복된다 | MES/CMMS는 기록 시스템, FieldSense는 작업 실행/검증 계층 |
| 구축이 오래 걸린다 | 1개 장비군/2개 SOP 파일럿로 시작 |

## 파일럿 제안 메시지

> 12주 동안 한 개 장비군과 두 개 절차에 적용해 작업 시간, 재작업, 안전 경고, 전문가 호출률을 baseline과 비교하겠습니다. 원본 영상은 저장하지 않고, 마스킹된 evidence와 작업 로그만 보관하는 방식으로 시작할 수 있습니다.
