# XR 사용자 경험 워크플로우

## 기본 작업 흐름

1. 작업자 로그인: SSO/QR/패스키/현장 배지
2. 작업 오더 선택: CMMS/MES 연동 또는 수동 선택
3. 장비 인식: QR/마커/객체 인식/위치 기반 asset 확인
4. 사전 안전 확인: PPE, LOTO, 주변 위험, 도구 준비
5. 단계별 안내: 음성 + 공간 카드 + 필요 위치 하이라이트
6. 단계 완료 검증: 시야 증거 + 손동작 + 작업자 확인
7. 예외 처리: 에러코드/부품 불일치/위험 감지 시 분기
8. 전문가 에스컬레이션: AI 예비 진단 + 현장 시야 공유
9. 작업 완료: 증거 로그, 서명, 리포트, 다음 정비 추천

## 대표 음성 명령

| 명령 | 시스템 동작 |
|---|---|
| “다음 단계 알려줘” | SOP state machine에서 다음 단계 표시 |
| “이 밸브 잠갔나?” | 시야의 밸브 상태 + SOP 조건 검증 |
| “매뉴얼 37쪽 절차 요약해줘” | 문서 검색 후 현재 작업 맥락에 맞게 요약 |
| “위험 요소 확인” | PPE/LOTO/고온/고압/전기 위험 checklist 실행 |
| “전문가 호출” | 현재 context bundle 생성 후 원격 전문가 연결 |
| “증거 저장” | 이미지/계기값/음성메모/단계 정보 저장 |

## UI 상태

| 상태 | 색/표현 방향 | 동작 |
|---|---|---|
| NORMAL | 중립 카드 | 다음 작업 안내 |
| CAUTION | 주의 카드 + 진동/음성 | 재확인 요청 |
| BLOCKED | 강한 경고 + 화면 중심 알림 | 작업 중지, 승인 필요 |
| EXPERT | 원격 전문가 오버레이 | 시야 공유/주석 |
| OFFLINE | 제한 모드 표시 | 캐시 절차만 허용 |

## 현장 UX 원칙

- 긴 문장을 피하고 “행동 단위”로 안내한다.
- 작업자의 시야를 가리지 않도록 peripheral overlay를 기본값으로 둔다.
- 소음 환경을 고려해 텍스트/아이콘/진동 피드백을 병행한다.
- 장갑 착용 상태에서 손 제스처 실패 가능성을 고려해 음성/시선/버튼 대체 입력을 제공한다.
- 위험 경고는 오탐도 문제지만 미탐이 더 큰 문제이므로 고위험 작업은 보수적으로 설계한다.
