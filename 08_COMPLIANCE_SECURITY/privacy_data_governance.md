# 개인정보와 데이터 거버넌스

## 원칙

1. 최소 수집: 추론에 필요한 데이터만 처리
2. 엣지 우선: 영상/음성 원본은 현장 내 처리
3. 기본 비저장: 원본 영상/음성 저장은 opt-in
4. 목적 제한: 교육/안전/품질 목적 외 사용 금지
5. 투명성: 작업자에게 수집 항목과 목적 고지
6. 접근 통제: 직무와 현장에 따른 최소 권한
7. 보존 제한: evidence별 retention policy 적용

## 데이터 처리 모드

| 모드 | 설명 | 권장 대상 |
|---|---|---|
| Redacted Evidence Only | masking된 이미지와 메타데이터만 저장 | 기본값 |
| Metadata Only | 시각 결과와 로그만 저장 | 개인정보 민감 현장 |
| Full Evidence Clip | 짧은 영상 저장 | 고위험/감사 필수 작업 |
| On-Prem Restricted | 고객망 내부 저장 | 의료/국방/핵심공정 |

## 작업자 고지 예시

```text
이 XR 시스템은 작업 안전과 품질 확인을 위해 작업 중 음성 명령, 장비 시야 이미지, 단계 완료 로그를 처리합니다. 원본 영상은 기본 저장되지 않으며, 안전/품질 증거가 필요한 경우 마스킹된 이미지와 메타데이터만 보관됩니다.
```

## 데이터 권한

- 작업자: 본인 세션 기록 열람 제한
- Supervisor: 소속 현장 작업 로그
- EHS/Quality: 안전/품질 evidence와 통계
- Admin: 사용자/기기/정책 관리
- Auditor: 감사 범위 내 read-only
- Vendor support: 고객 승인 session만 접근

## 데이터 삭제/보존

- 개인식별 가능 데이터는 정책 기간 이후 자동 삭제
- 감사 증거는 법/고객 정책 기간 유지
- 모델 개선 데이터는 별도 동의와 비식별 처리 후 사용
- 고객 계약 종료 시 데이터 export/delete 절차 제공
